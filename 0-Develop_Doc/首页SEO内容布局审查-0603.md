## 审查意见：首页 SEO 内容布局优化

**审查对象：** `/src/app/[locale]/page.tsx` 及 SEO 组件
**审查日期：** 2026-06-03
**审查人：** 素贞

### 一、问题描述

| 编号 | 问题类型             | 问题描述                                                     | 严重程度 |
| ---- | -------------------- | ------------------------------------------------------------ | -------- |
| P1   | 视觉单调             | 三个卡片区块（HowToUse、UseCases、HowItWorks）布局结构完全相同，缺乏视觉层次和辨识度 | 高       |
| P2   | 宽度不一致           | SEO 内容区 `.seo-content .container` 限制为 `max-width: 700px`，而工具区为 `--max-tool: 960px`，造成视觉断层 | 中       |
| P3   | 区块顺序不合理       | 当前顺序：HowToUse → UseCases → HowItWorks → Why → FAQ。原理说明应前置，价值主张应合并 | 中       |
| P4   | Why Section 过于简陋 | 仅一段文字，与上方卡片区块视觉落差大，核心价值主张不够突出   | 高       |
| P5   | 缺少视觉焦点         | 无数字标注、无 icon 辅助，卡片平铺无重点引导                 | 中       |

### 二、修改要求

#### 2.1 统一宽度

**文件：** `src/app/globals.css`

**修改：**

```css
/* 修改前 */
.seo-content .container {
  max-width: 700px;
}

/* 修改后 */
.seo-content .container {
  max-width: var(--max-tool); /* 960px，与工具区一致 */
}
```

---

#### 2.2 调整区块顺序

**文件：** `src/app/[locale]/page.tsx`

**修改后顺序：**
1. HowItWorks（原理说明，建立信任）
2. HowToUse（使用步骤，降低门槛）
3. UseCases（使用场景，展示价值）
4. FAQ（常见问题，消除疑虑）
5. Bottom CTA

**删除：** Why Section（内容合并到 UseCases 开头）

---

#### 2.3 HowItWorks 区块优化

**要求：**
- 保持 3 列网格布局
- 每个卡片顶部添加**数字圆圈**（1/2/3），样式：`w-10 h-10 rounded-full bg-primary text-white flex items-center justify-center font-bold
text-lg`
- 卡片添加微妙的 hover 效果：`hover:border-primary hover:shadow-sm transition`

**参考结构：**

```tsx
<div className="grid gap-6 md:grid-cols-3">
  {items.map((item, i) => (
    <div key={i} className="border rounded-xl p-6 hover:border-primary hover:shadow-sm transition">
      <div className="w-10 h-10 rounded-full bg-primary text-white flex items-center justify-center font-bold text-lg mb-4">
        {i + 1}
      </div>
      <h3 className="text-lg font-semibold mb-2">{item.title}</h3>
      <p className="text-muted-foreground text-sm">{item.desc}</p>
    </div>
  ))}
</div>
```

---

#### 2.4 HowToUse 区块优化

**要求：**
- 改为**横向步骤流程**布局（移动端堆叠）
- 使用**连接线**或箭头连接步骤
- 步骤标签 `Step 1/2/3` 改为**醒目的数字徽章**

**参考结构：**

```tsx
<div className="flex flex-col md:flex-row gap-6 items-start">
  {steps.map((step, i) => (
    <div key={i} className="flex-1 text-center">
      <div className="w-12 h-12 rounded-full bg-accent text-primary flex items-center justify-center font-bold text-xl mx-auto mb-4">
        {i + 1}
      </div>
      <h3 className="font-semibold mb-2">{step.title}</h3>
      <p className="text-sm text-muted-foreground">{step.desc}</p>
      {i < steps.length - 1 && (
        <div className="hidden md:block absolute top-6 left-full w-full h-px bg-border -translate-x-1/2" />
      )}
    </div>
  ))}
</div>
```

---

#### 2.5 UseCases 区块优化

**要求：**
- 合并 Why Section 的内容作为区块开头的**价值主张段落**
- 保持 2 列网格
- 每个卡片添加**icon 占位**（后续可替换为实际 icon）
- 第一张卡片使用**主色边框**突出：`border-primary`

**参考结构：**

```tsx
<section className="seo-section">
  <h2>{t('seoUseTitle')}</h2>
  <p className="text-center text-muted-foreground max-w-2xl mx-auto mb-8">
    {/* 原 Why Section 内容 */}
    {t('seoWhyDesc')}
  </p>
  <div className="grid gap-6 md:grid-cols-2">
    {cases.map((item, i) => (
      <div key={i} className={cn(
        "border rounded-xl p-6",
        i === 0 && "border-primary"
      )}>
        <div className="w-10 h-10 rounded-lg bg-accent flex items-center justify-center mb-4">
          {/* icon placeholder */}
          <span className="text-primary text-xl">💡</span>
        </div>
        <h3 className="font-semibold mb-2">{item.title}</h3>
        <p className="text-sm text-muted-foreground">{item.desc}</p>
      </div>
    ))}
  </div>
</section>
```

---

#### 2.6 FAQ 区块保持不变

当前手风琴交互已足够，无需修改。

### 三、验收标准

| 检查项      | 期望结果                               |
| ----------- | -------------------------------------- |
| 宽度一致性  | SEO 区与工具区宽度相同（960px）        |
| 区块顺序    | HowItWorks → HowToUse → UseCases → FAQ |
| HowItWorks  | 每个卡片顶部有数字圆圈（1/2/3）        |
| HowToUse    | 横向步骤布局，步骤之间有连接线或箭头   |
| UseCases    | 包含价值主张段落，第一张卡片有主色边框 |
| Why Section | 已删除，内容合并到 UseCases            |
| 响应式      | 移动端布局正确（堆叠而非挤成一团）     |
| hover 效果  | 卡片 hover 时有边框/阴影变化           |
