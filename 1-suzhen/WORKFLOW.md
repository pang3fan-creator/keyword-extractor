# 项目工作流程

## 文件夹结构

```
project-name/
├── 1-suzhen/                      # 专属文件夹
│   ├── KEYWORD-RESEARCH.md        # 关键词研究
│   ├── COMPETITOR-ANALYSIS.md     # 竞品分析
│   ├── PROGRESS.md                # 项目进度
│   └── ...                        # 其他讨论笔记
└── (Claude Code 生成的项目文件)    # 代码实现
```

---

## 分工原则

### 素贞的职责

- ✅ 关键词研究、数据分析
- ✅ 竞品分析、市场调研
- ✅ 提出建议、给出意见
- ✅ 记录讨论过程、决策记录
- ✅ 项目进度跟踪
- ❌ **不修改具体代码**
- ❌ **不做工程实现**

### Claude Code 的职责

- ✅ 具体的工程实现
- ✅ 代码编写、修改
- ✅ 功能开发、测试

### 主人的职责

- ✅ 做决策
- ✅ 在 Claude Code 上修改代码
- ✅ 最终审核

---

## 固定流程

1. **开始项目** → 素贞在 `own/` 新建项目文件夹
2. **创建 `1-suzhen/` 文件夹** → 存放素贞的笔记
3. **讨论内容保存** → 所有有价值信息保存到 `1-suzhen/`
4. **工程实现** → Claude Code 在项目根目录生成代码
5. **推送 GitHub** → 定期同步

---

## 配合原则

1. **素贞只提意见，不写代码**
2. **讨论记录保存在 `1-suzhen/`，与代码分离**
3. **主人看完讨论后，用 Claude Code 实现具体功能**
